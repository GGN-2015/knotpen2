# 项目算法手册

[English version](./algorithm-manual.md)

本文说明 Knotpen2 使用的主要算法。

## 范围

Knotpen2 处理的是用户绘制的扭结或链环平面投影图。节点表示投影曲线上的控制点，边连接相邻控制点。交叉点不会作为节点保存，而是通过边之间的几何相交实时检测得到。

核心实现位于：

- `knotpen2/MyAlgorithm.py`
- `knotpen2/MemoryObject.py`
- `knotpen2/math_utils.py`

## PD_CODE 计算前置条件

计算 PD_CODE 前，程序会检查：

1. 至少有三个节点。
2. 每个节点的度恰好为 2。
3. 每个连通分支恰好有一个起始点。
4. 每个连通分支恰好有一个方向点。
5. 每个连通分支的起始点和方向点相邻。

程序目前不会自动检测所有几何非法情况。用户必须人工保证没有三线共点。

## 图模型

图形被表示为无向图：

- `dot_dict` 保存节点位置。
- `line_dict` 保存节点之间的边。
- `degree` 保存每个节点的图论度数。

因为合法图中每个节点的度都为 2，所以每个连通分支会被视为一个闭合折线环。

## 连通分支

`MyAlgorithm.get_adj_list()` 根据 `line_dict` 构造邻接表。

`MyAlgorithm.get_connected_components()` 在邻接表上执行 DFS，返回：

- `adj_list`：节点邻接关系。
- `block_list`：连通分支列表，每个分支由节点 ID 列表表示。

## 定向

每个连通分支必须有：

- 一个起始点。
- 一个与起始点相邻的方向点。

`solve_pd_code()` 会旋转并在必要时反转每个连通分支，使其满足：

```text
component[0] == 起始点
component[1] == 方向点
```

这样每个连通分支都有确定的遍历方向。

## 交叉点检测

程序会枚举每一对无序边来检测交叉。

如果两条边共享端点，则跳过。否则使用 `math_utils.segments_intersect()` 计算交点以及交点在两条线段上的参数 `t1`、`t2`。

只有严格位于两条线段内部的交叉会被接受：

```text
0 < t1 < 1
0 < t2 < 1
```

每个交叉点记录：

- 交点坐标。
- 两条相交边分别属于哪个连通分支、哪一段。
- 交点在局部线段上的参数。
- 两条边的 ID。

## 上下关系

交叉点的上下关系保存在 `MemoryObject.inverse_pairs` 中。

`MemoryObject.check_line_under(line_id_1, line_id_2)` 用来判断某条边是否在另一条边下方。默认上下关系由边编号的数值大小决定，`inverse_pairs` 用于反转默认关系。

用户在界面中点击交叉点即可切换这个关系。

## 弧段拆分

检测到交叉点后，每个连通分支会被拆分为弧段。

对每个交叉点，算法生成两个半交叉记录，分别放在涉及的两个连通分支上。然后按以下键排序：

```text
(连通分支中的节点序号, 在线段上的局部参数)
```

排序后的半交叉点会把连通分支切分为若干弧段。每个弧段之后会得到用于最终 PD_CODE 的全局整数编号。

## PD_CODE 构造

对于每个交叉点，算法会：

1. 判断哪条线在下方，哪条线在上方。
2. 找到交叉点周围的进入弧和离开弧编号。
3. 使用二维左转测试确定局部顺序。
4. 生成包含四个弧编号的原始交叉点记录。

原始弧编号是连通分支内部的元组。算法会收集并排序所有唯一弧编号，再映射为从 1 开始的最终整数标签。

最终显示的 PD_CODE 是排序后的四元组列表。

## SVG 生成

`MyAlgorithm.calculate_svg()` 使用连通分支遍历顺序和交叉拆分数据生成 SVG 路径。

关键细节：

- 曲线以二次贝塞尔路径输出。
- 下穿线会在交叉点附近缩短，从视觉上表现上下关系。
- `.num.svg` 包含弧线编号文字。
- `.nonum.svg` 不包含弧线编号。
- `.arrow.svg` 包含方向箭头。

SVG 的 viewBox 根据节点包围盒计算。

## 已知限制

- 程序不会自动拒绝三线共点。
- 存档格式默认信任 `line_dict` 和 `degree` 之间的一致性。
- 图形很密集时可能较慢，因为交叉检测会枚举边对。
- 上下关系绑定在边对上，因此移动节点后交叉点集合可能变化。移动节点后应重新检查交叉点上下关系。
